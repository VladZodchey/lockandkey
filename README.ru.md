## Сборка из исходников
```shell
git clone https://github.com/VladZodchey/lockandkey.git lockandkey && cd lockandkey
uv venv  # Предполагает, что у вас уже стоит uv
uv sync
uv build
```


## Стэк

Этот проект использует:
- QT (Через Python-модуль `PyQT6`) (Интерфейс)
- Material Design 3 (А именно `/src/resources/icons`) (комплект иконок)
- SQLite (Через встроенный Python-модуль `sqlite3`) (Структура базы данных)
- AES256 + PBKDF (Через Python-модуль `cryptography`) (Шифрование)
- QR-Коды (Через Python-модуль `qrcode`) (Распространение ссылки)

Для разработки применялись:
- `ruff` (линтер кода)
- `uv` (менеджер пакетов)
- `pyinstaller` (билд исполняемых файлов)
- QT Designer (создание виджетов)

## Зачем его использовать? Есть же KeePass...

Не используйте. Это -- проект для портфолио. У него много недостатков, странных дизайнерских решений.
KeePass проверен временем. KeePassXC, его форк, получает поддержку сообществом и следует современным стандартам безопасности.
Оба имеют открытый код, у обоих гораздо больше возможностей чем у этой поделки.
